# Turnover Inspection Checklist — Usage Examples
**Owner:** Arthur F. Appling Sr. — AA Capital Inc. / Prime Pathwy

---

## Example 1 — Full Turnover (All 10 Points)

**Scenario:** 2-bedroom unit, post-eviction. Scope: haul-out, deep clean, paint, final clean.

**Sequence:**
1. Complete Job Header — address, unit, Job Type = Full Turnover, crew lead name
2. **Point 1** — Before photos: every room, all debris, all damage, exterior. Upload to vault.
3. **Point 2** — Haul-out: remove all furniture and debris. Dump run. Receipt photographed and uploaded.
4. **Point 3 + 4 + 5** — Deep clean kitchen, bathrooms, all rooms in sequence.
5. **Point 6** — Paint prep: patch holes, prime scuffs, tape trim and fixtures.
6. **Point 7** — Paint: two full coats, trim touched up, no drips.
7. **Point 8** — Final clean: remove tape, clean overspray, mop all floors, clean windows.
8. **Point 9** — After photos: every room at matching angles, repaired areas close-up. Upload to vault.
9. **Point 10** — Supervisor walkthrough, punch list resolved, client walkthrough, written sign-off obtained.

**Expected Result:** All 10 points marked PASS. OVERALL VERDICT: PASS — JOB CLOSED.

---

## Example 2 — Clean-Out Only

**Scenario:** Unit cleared by tenant. Scope: debris haul-away and deep clean only. No paint.

**Active Points:** 1, 2, 3, 4, 5, 9, 10. Points 6, 7, 8 are N/A — mark as such.

**Key checkpoint:** Point 2 disposal receipt is mandatory even on a clean-out. No receipt = expense not billable.

---

## Example 3 — Disputed Job Documentation

**Scenario:** Property manager claims work was not completed. Job was done 3 weeks ago.

**Response Protocol:**
```
1. Retrieve from vault/[CLIENT]/photos/before/ — timestamped before photos
2. Retrieve from vault/[CLIENT]/photos/after/ — matching after photos
3. Retrieve from vault/[CLIENT]/reports/ — signed checklist
4. Retrieve from vault/[CLIENT]/receipts/ — disposal receipt if applicable
5. Produce documentation package: before/after + signed checklist + receipt
```

**Expected Result:** Dispute resolved by documentation. No verbal argument required. Institutional Grade protection.

---

## Anti-Patterns

- **DO NOT** start work before completing Point 1 (before photos) — any area worked without before photos is undefendable in a dispute
- **DO NOT** accept a verbal sign-off for Point 10 — email confirmation is the minimum; signed form is preferred
- **DO NOT** mark a point PASS before the standard is physically met — "in progress" is not PASS
- **DO NOT** close a job with an open punch list — return visit must be scheduled before marking PASS — JOB CLOSED
- **DO NOT** upload photos to personal storage — all photos go to `vault/[CLIENT]/photos/` only
