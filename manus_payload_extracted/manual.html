Execution Tool: execute_workflow.py

To bridge the gap between the Markdown workflows and actual execution, the system utilizes a standardized Python tool. This tool reads the designated workflow file, displays the execution parameters, and prompts the operator to verify the outputs against the Pass Criteria. It is the mechanical execution layer of the WAT Framework.

File: /tools/execute_workflow.py #!/usr/bin/env python3
"""
Tool: execute_workflow.py
Purpose: Parse and execute Markdown-based workflows within the Prime Pathwy Sovereign System.
Architect: Arthur F. Appling Sr.
"""

import sys, os

def run_workflow(workflow_path):
    if not os.path.exists(workflow_path):
        print(f"ERROR: Workflow {workflow_path} not found.")
        sys.exit(1)
    print(f"Initializing execution for: {workflow_path}")
    print("=" * 60)
    with open(workflow_path, 'r') as f:
        print(f.read())
    print("=" * 60)
    print("Execution complete. Review outputs against Pass Criteria.")

if __name__ == "__main__":
    if len(sys.argv) < 2:
        print("Usage: python3 execute_workflow.py <path_to_workflow.md>")
        sys.exit(1)
    run_workflow(sys.argv[1])

Usage

Exact Command python3 /tools/execute_workflow.py /workflows/01_institutional_knowledge_vault.md Pass Criteria Terminal displays the full workflow content followed by "Execution complete." Error Map If "Workflow not found" error, verify the absolute path to the workflow file is correct.
