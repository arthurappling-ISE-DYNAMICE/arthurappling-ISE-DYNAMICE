01
Section One
Understanding Safe vs. Dangerous Actions
Safe Actions
  • Opening files
  • Reading CSV data
  • Running display commands
  • Copying folders
  • Creating backups
  • Viewing workflows
Dangerous Actions
  • Deleting folders
  • Renaming system directories
  • Overwriting CSV databases
  • Bulk editing without backups
  • Running unknown scripts
  • Moving folders manually
Standing Rule

If you are unsure what a command does — do not run it.

If the Terminal Says 'Path Not Found'
⚑ Symptom

The terminal cannot locate your PrimePathwyOS folder.

◉ Common Causes
  • Folder moved accidentally
  • Typing error in command
  • Wrong username inside path
  • Folder renamed
▶ Recovery Process
1

Open File Explorer

2

Navigate manually to C:\Users\arthu\GeminiEcosystem\

3

Confirm PrimePathwyOS still exists

4

Copy the exact folder path

5

Paste the copied path into PowerShell

Correct Command
cd "C:\Users\arthu\GeminiEcosystem\PrimePathwyOS"
Important

Paths must match exactly. One wrong character breaks the entire command.

Section Three
If the CSV Database Will Not Open
⚑ Symptom

The CRM database fails to load or is inaccessible.

◉ Common Causes
  • File renamed
  • File moved
  • CSV corrupted
  • Wrong directory

Navigate to confirm file existence:

research\leads\enriched_solano_crm_database.csv
▶ Safe Recovery
  • Restore from backup copy
  • Re-download the latest version
  • Open in Excel to confirm readability
✕ Dangerous Action

Do not save corrupted CSV exports over the master file.

04
Section Four
If PowerShell Commands Fail
⚑ Symptom

Commands return red text or syntax errors.

◉ Common Causes
  • Missing quotation marks
  • Missing spaces
  • Incorrect slashes
  • Pasting partial commands

Recovery method:

1
Compose yourself

Do not panic. Red text is almost always a formatting problem, not a system failure.

2
Re-copy

Re-copy the full original command from your source document.

3
Paste exactly

Paste it exactly as written — no partial edits.

4
One at a time

Run one command at a time. Never chain unknowns.

05
Section Five
If You Accidentally Delete Something
1
Stop immediately

Do not continue moving or editing files.

2
Check Recycle Bin

Navigate to the Recycle Bin immediately.

3
Restore if available

Right-click the file and select Restore.

▶ If File Is Not in Recycle Bin
  • Check cloud backups
  • Check external drive backups
  • Search entire drive by filename
Prevention Rule

Never operate without backups. This situation should be impossible.
