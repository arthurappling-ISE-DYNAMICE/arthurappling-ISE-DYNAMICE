Safe vs. Dangerous Actions
▾
Status:
Safe Actions
  • Opening files
  • Reading CSV data
  • Running display commands
  • Copying folders
  • Creating backups
  • Viewing workflows
Dangerous Actions
  • Deleting folders
  • Renaming system directories
  • Overwriting CSV databases
  • Bulk editing without backups
  • Running unknown scripts
  • Moving folders manually
Standing Rule

If you are unsure what a command does — do not run it.

▾

⚑ Symptom

The terminal cannot locate your PrimePathwyOS folder.

◉ Common Causes
  • Folder moved accidentally
  • Typing error in command
  • Wrong username inside path
  • Folder renamed
▶ Recovery Process
1

Open File Explorer

2

Navigate manually to C:\Users\arthu\GeminiEcosystem\

3

Confirm PrimePathwyOS still exists

4

Copy the exact folder path

5

Paste the copied path into PowerShell

Correct Command

Important

Paths must match exactly. One wrong character breaks the entire command.

Troubleshooting · Section 4
If PowerShell Commands Fail
▾
Status:
⚑ Symptom

Commands return red text or syntax errors.

◉ Common Causes
  • Missing quotation marks
  • Missing spaces
  • Incorrect slashes
  • Pasting partial commands
1
Do Not Panic

Red text is almost always a formatting problem, not a system failure.

2
Re-Copy Full Command

Go back to the source document and copy the full original command.

3
Paste Exactly

Paste it exactly — no partial edits or manual typing.

4
Run One at a Time

Never chain unknown commands in sequence.

Troubleshooting · Section 5
If You Accidentally Delete Something
▾
Status:
1
Stop Immediately

Do not continue moving or editing any files.

2
Check Recycle Bin

Navigate to the Recycle Bin immediately — do not delay.

3
Restore If Available

Right-click the deleted file and select Restore.

▶ If File Is Not in Recycle Bin
  • Check cloud backups immediately
  • Check external drive backups
  • Search entire drive by filename
Prevention Rule

Never operate without backups. This situation should be impossible.
