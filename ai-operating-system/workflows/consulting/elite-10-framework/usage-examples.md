## Example 1 — Paperwork Trap Client

**Scenario:** Hauling operator, $200K/year revenue, spends 3 hours/night on invoices and estimates. Has 2 drivers but is doing all admin himself.

**Diagnosis result:** Paperwork trap → Doc Processing Engine first.

**Phase 2 Install sequence:**
1. Master Ledger logic (revenue vs. expense categorization)
2. Digital receipt vault folder structure
3. Project Audit Log template from `/Prime`
4. Before/after photo checklist for every job

**Pass criteria target:** Client produces clean audit package for any job within 24 hours. Nightly admin drops to under 30 minutes.

**Close trigger:** "$200K revenue, 3 hours/night in admin = $30,000/year in owner time. $5,000 to eliminate it — does that make sense?"

## Example 2 — Bid Starvation Client

**Scenario:** Janitorial operator, $120K/year, has SB certification, never knows where the next contract is coming from. Relies entirely on word of mouth.

**Diagnosis result:** Bid starvation → Speed to Lead Engine first.

**Phase 2 Install sequence:**
1. SAM.gov / Cal eProcure daily digest alert setup
2. Pre-written bid response templates for NAICS 561720
3. Master Ledger bid tracking (open → submitted → awarded → closed)
4. 48-hour turnaround protocol wired to their inbox

**Pass criteria target:** Client submits 2+ government bids/month with zero manual SAM.gov search.

## Example 3 — Client Disqualification

**Scenario:** Logistics operator, $35K/year revenue (below $50K threshold), wants "a quick system that gets me more jobs."

**Disqualification:**
```
Revenue: $35K — below $50K threshold
Mindset: "quick system" = not committed to 12-month transformation
Decision: DISQUALIFY

Response: "The system I install takes 12 months to build correctly and requires full documentation
buy-in from day one. At your current revenue stage, the ROI math doesn't support a $5,000 install.
Come back when you're at $80K — I'll still be here."
```

No pitch. No close attempt. Referral to appropriate resource if possible.

## Anti-Patterns

- **DO NOT** prescribe the engine before the client names their bottleneck — diagnosis comes first, always
- **DO NOT** attempt the close before all three phases are presented — the close is the final question, not the opening
- **DO NOT** take a client under $50K annual revenue — the ROI math does not support a system install at that stage
- **DO NOT** accept a client who is unwilling to document — the system requires owner buy-in to hold
- **DO NOT** run more than 10 active engagements simultaneously — quality degrades above this threshold
